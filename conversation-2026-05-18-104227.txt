  ---
  What Is Claude Local Bridge?

  Claude Local Bridge is a VS Code extension that does one core thing: it turns
  your existing Claude Code subscription into a local HTTP API server that any
  tool on your machine can talk to.

  Normally, tools like Cursor, Continue.dev, or a custom script need their own
  Anthropic API key to reach Claude. This extension eliminates that requirement.
   It quietly watches what Claude Code is already doing вҖ” it's already
  authenticated, already talking to Anthropic вҖ” and exposes that same access via
   a local server at http://localhost:11437.

  Clients can speak either the Anthropic Messages API format or the OpenAI Chat 
  Completions format. The extension handles the translation automatically.

  The most architecturally elegant part of this project is how it avoids asking
  you for credentials at all. Rather than prompting for an API key, it
  monkey-patches Node's https.request() and the global fetch() function. Every
  time Claude Code makes a request to api.anthropic.com, the interceptor
  silently reads the auth header off the outgoing request and saves it. This
  means credentials are always fresh вҖ” they automatically update whenever Claude
   Code refreshes its OAuth token.

  ---
  Request Flow: Step by Step

  Here is what happens from the moment your local tool sends a request, to the
  moment it gets a response back.

  Step 1 вҖ” Your client sends a request

  Your tool (Cursor, a custom script, etc.) makes an HTTP request to
  http://localhost:11437. It can use either format:

  - POST /v1/messages вҖ” Anthropic's native format
  - POST /v1/chat/completions вҖ” OpenAI's format

  Step 2 вҖ” src/server.js receives and routes it

  The Node.js HTTP server is listening. It reads the Content-Type, adds CORS
  headers (only localhost origins are allowed), reads the request body, and
  dispatches to the right handler function based on the URL path.

  Step 3 вҖ” The handler parses and normalizes the request

  If you sent Anthropic format (src/handlers/anthropic.js): the request is
  parsed, the model name is resolved through the alias table (src/models.js),
  and max_tokens is defaulted to 4096 if missing.

  If you sent OpenAI format (src/handlers/openai.js): the request goes through
  openAIToAnthropic() in the translator. This converts OpenAI's message array
  into Anthropic's format вҖ” pulling out system messages, converting tool call
  structures, converting image blocks, and remapping stop reasons.

  вҳ… Insight в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  The OpenAI-to-Anthropic translation is non-trivial. In OpenAI's format, tool
  messages are separate entries in the messages array. Anthropic represents tool
   results as tool_result content blocks nested inside a user message. The
  translator walks the messages array and restructures these correctly. It also
  handles image data URIs вҖ” base64-encoded images are detected and converted to
  Anthropic's {type: "image", source: {type: "base64", ...}} block format.
  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  Step 4 вҖ” Credentials are discovered

  src/credentials.js runs a priority chain to find valid credentials. It checks
  these sources in order, stopping at the first one that succeeds:

  1. Intercepted token вҖ” pulled from Claude Code's live outgoing requests
  (highest priority, always fresh)
  2. ANTHROPIC_API_KEY environment variable
  3. CLAUDE_CODE_OAUTH_TOKEN environment variable
  4. macOS Keychain вҖ” runs security find-generic-password -s 'Claude 
  Code-credentials' -w and parses the JSON
  5. ~/.claude/.credentials.json вҖ” a credential file on disk (works on
  Linux/Windows too)
  6. VS Code setting claudeLocalBridge.apiKey вҖ” a manual fallback (lowest
  priority)

  Results are cached for 5 minutes, so the keychain or file isn't read on every
  request.

  Step 5 вҖ” Auth headers are built

  src/fingerprint.js builds the headers for the upstream Anthropic request. In
  the default compatibility identity mode, it replays the exact headers that
  were captured from Claude Code's requests вҖ” including user-agent,
  anthropic-version, anthropic-beta, and Stainless SDK headers. This makes the
  proxied request look identical to a real Claude Code request.

  If no live fingerprint has been captured yet, it falls back to hardcoded
  defaults.

  Step 6 вҖ” src/proxy.js forwards the request upstream

  proxyToAnthropic() opens a real HTTPS connection to api.anthropic.com (or
  whatever hostname was intercepted from Claude Code), sends the transformed
  request body, and starts piping the response back.

  If the response is streaming (SSE), the bytes are piped directly to your
  client without buffering. If you sent an OpenAI format request, the SSE events
   are intercepted by a stream converter that translates each
  content_block_delta Anthropic event into the OpenAI choices[0].delta.content
  shape on the fly.

  If Anthropic returns a 401 Unauthorized, the credential cache is cleared and
  the request is retried exactly once with freshly discovered credentials.

  Step 7 вҖ” Your client receives the response

  For streaming responses, your client sees a live stream of Server-Sent Events.
   For non-streaming, it gets a single JSON response. Either way, the format
  matches whatever the client originally asked for вҖ” Anthropic or OpenAI вҖ”
  without the client needing to know which upstream API was actually used.

  ---
  How the HTTPS Interceptor Works

  This is the cleverest part of the extension. When the extension activates,
  src/interceptors/https.js does this:

  // Save the real function
  ctx._originalHttpsRequest = https.request;

  // Replace it with our wrapper
  https.request = function interceptedRequest(optionsOrUrl, optionsOrCb, 
  ...rest) {
    // Look at the target host вҖ” is this going to Anthropic?
    if (host is in ['api.anthropic.com', 'claude.ai', 'api.claude.ai']) {
      // Read the auth header and save it to ctx
      captureToken(headers);
      captureFingerprint(headers);
    }
    // ALWAYS call the real https.request вҖ” never break anything
    return ctx._originalHttpsRequest.call(this, optionsOrUrl, optionsOrCb,
  ...rest);
  };

  The same thing is done for globalThis.fetch, because the Anthropic SDK now
  uses the browser-style Fetch API internally. Both must be patched for full
  coverage.

  The interceptor is purely passive вҖ” it never modifies or delays any request.
  It only reads.

  How Streaming Works End-to-End

  Anthropic's streaming API sends Server-Sent Events (SSE) вҖ” a sequence of text
  frames like:

  event: content_block_delta
  data: {"type":"content_block_delta","index":0,"delta":{"type":"text_delta","te
  xt":"Hello"}}

  event: message_stop
  data: {"type":"message_stop"}

  If a client requested OpenAI-format streaming, the bridge intercepts these
  frames and converts them on-the-fly via
  createAnthropicToOpenAIStreamConverter(). Each incoming chunk is:

  1. Accumulated in a small buffer
  2. Split on \n\n boundaries to isolate individual SSE frames
  3. Parsed to extract the data: JSON
  4. Converted to the OpenAI chunk shape: {"choices": [{"delta": {"content": 
  "Hello"}}]}
  5. Sent to the client immediately as data: <json>\n\n

  There is no full-message buffering вҖ” the client gets tokens as they arrive,
  giving true real-time streaming.
